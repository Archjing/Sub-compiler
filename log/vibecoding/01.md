### 二、 引入 AST 概念的结构化重构思路

在编译原理中，AST（Abstract Syntax Tree，抽象语法树）的作用是将“源代码的文本形式”转化为“剥离了语法糖的纯粹结构”。

在字幕处理的领域，**我们不需要生成编程语言级别的 AST，而是需要建立一个“字幕领域的通用中间表示（Intermediate Representation, IR）”**。你的 `dict` 列表已经是 IR 的雏形，但我们可以用面向对象的思想将其变成真正的“数据模型”。

重构的核心思想是：**输入源 (Parser) -> 标准通用模型 (AST) -> 渲染器 (Generator/Writer)**。

#### 1. 定义字幕的 AST 模型 (Model)

不再使用松散的字典，而是定义严谨的数据类（Data Class）。这不仅能利用类型提示（Type Hint），还能把时间计算封装在内部。

- **`Timecode` 类**：专门处理时间的解析、秒数加减、格式化输出。
- **`SubtitleCue` 类**：代表单一的一句字幕（节点）。包含：起始时间、结束时间、文本内容、甚至未来的样式元数据。
- **`SubtitleDocument` 类**：代表整个字幕文件（树的根节点）。包含：Cue 列表、全局元数据（如语言、标题）。

#### 2. 将解析器 (Parsers) 变成 AST Builder

解析器的唯一任务，就是读取不管多么烂的格式，将其转换为我们定义的 `SubtitleDocument` 对象。

- 可以引入**策略模式 (Strategy Pattern)**，定义一个 `BaseParser`，然后让 `JsonParser`, `LrcParser`, `RegexTextParser` 继承它。
- 这样可以把正则匹配的脏活累活限制在特定的 Parser 类里，防止污染全局。

#### 3. 将输出函数变成 AST Generators

生成器只接收 `SubtitleDocument` 对象，不关心数据是从 JSON 来的还是从 LRC 来的。

- 定义 `BaseWriter`，然后实现 `SrtWriter`, `JsonWriter` 等。
- 生成器直接调用 AST 节点自身的方法（如 `cue.get_srt_time()`）来获取需要的格式，减少字符串操作。

#### 4. 健壮性提升 (Robustness)

- **编码嗅探**：目前硬编码了 `utf-8`，对于国内常见的 GBK 编码 SRT 文件会直接崩溃。需要引入 `chardet` 或简单的 `try-except` fallback 机制。
- **异常隔离**：在解析单行失败时，记录 Warning 并继续下一行，而不是让整个程序崩溃。
- **解耦的业务逻辑**：将 CLI 参数解析（`argparse`）与核心转换逻辑（`Converter` 类）完全分离，这样未来如果想给它加个 GUI 界面，或者集成到 Web 后端，无需修改核心代码。

class Timecode:
    """时间戳对象，内部统一使用总毫秒数存储，彻底杜绝字符串计算错误"""
    def __init__(self, total_ms: int):
        self.total_ms = max(0, total_ms)

    @classmethod
    def from_string(cls, ts: str) -> 'Timecode':
        """将乱七八糟的字符串解析为标准的时间戳对象"""
        ts_clean = ts.lower().replace("h", ":").replace("m", ":").replace(",", ".")
        parts = ts_clean.split(":")
        
        try:
            if len(parts) == 2:  # mm:ss.ms
                h, m, s_str = 0, int(parts[0]), parts[1]
            elif len(parts) == 3: # hh:mm:ss.ms
                h, m, s_str = int(parts[0]), int(parts[1]), parts[2]
            else:
                return cls(0)
            
            if '.' in s_str:
                s_val, ms_str = s_str.split('.')
                s = int(s_val)
                ms = int(ms_str.ljust(3, "0")[:3])
            else:
                s, ms = int(s_str), 0
                
            total_ms = (h * 3600 + m * 60 + s) * 1000 + ms
            return cls(total_ms)
        except ValueError:
            return cls(0)

    def add_seconds(self, seconds: float) -> 'Timecode':
        return Timecode(self.total_ms + int(seconds * 1000))

    def to_standard(self, separator='.') -> str:
        """输出标准格式: HH:MM:SS.mmm (或根据separator变更为 HH:MM:SS,mmm)"""
        total_s = self.total_ms // 1000
        ms = self.total_ms % 1000
        h = total_s // 3600
        m = (total_s % 3600) // 60
        s = total_s % 60
        return f"{h:02d}:{m:02d}:{s:02d}{separator}{ms:03d}"

    def to_lrc_format(self) -> str:
        """输出 LRC 常用格式: mm:ss.xx"""
        total_s = self.total_ms // 1000
        ms = self.total_ms % 1000
        m = total_s // 60
        s = total_s % 60

        return f"{m:02d}:{s:02d}.{xx:02d}"

            for m in ts_matches:
                temp_cues.append({
                    "start": Timecode.from_string(m.group(1)),
                    "text": text
                })
        
        # LRC 缺失结束时间，需要根据上下文补全
        temp_cues.sort(key=lambda x: x["start"].total_ms)
        for i, cue_data in enumerate(temp_cues):
            start_tc = cue_data["start"]
            if i + 1 < len(temp_cues):
                end_tc = temp_cues[i+1]["start"]
            else:
                end_tc = start_tc.add_seconds(3.0) # 最后一句默认持续3秒
            doc.add_cue(start_tc, end_tc, cue_data["text"])

    @staticmethod
    def _parse_text(content: str, doc: SubtitleDocument):
        """状态机解析：兼容性极强的回退方案"""
        current_start = None
        current_end = None
        current_text = []

        def flush_cue():
            nonlocal current_start, current_end, current_text
            if current_start and current_end and current_text:
                doc.add_cue(current_start, current_end, "\n".join(current_text))
            current_start = current_end = None
            current_text = []

        for line in content.splitlines():
            line = line.strip()
            if not line or line.isdigit(): 
                continue # 忽略空行和 SRT 的纯数字序号行

            range_match = RANGE_PATTERN.search(line)
            if range_match:
                flush_cue() # 遇到新的时间轴，先把之前的存起来
                current_start = Timecode.from_string(range_match.group(1))
                current_end = Timecode.from_string(range_match.group(2))
                # 检查同一行时间轴后面是否跟着文本
                inline_text = line[range_match.end():].strip()
                if inline_text:
                    current_text.append(inline_text)
            elif current_start: # 只有在已经捕获了时间轴的情况下，才收集文本
                current_text.append(line)
        
        flush_cue() # 文件结束时清空缓冲区

# 3. 生成器 (Writer: AST -> Target Format)
# ==========================================

class WriterFactory:
    @staticmethod

        for i, cue in enumerate(doc.cues, 1):
            start_time = cue.start.to_standard(separator=',') # SRT 规范：逗号分隔毫秒
            end_time = cue.end.to_standard(separator=',')
            clean_text = cue.text.strip()
            file_obj.write(f"{i}\n{start_time} --> {end_time}\n{clean_text}\n\n")

    @staticmethod
    def _write_sbv(doc: SubtitleDocument, file_obj):
        """严格遵守 YouTube SubViewer (.sbv) 规范的输出逻辑"""
        for cue in doc.cues:
            # SBV 规范：点号分隔毫秒，时间轴之间仅用逗号连接
            start_time = cue.start.to_standard(separator='.')
            end_time = cue.end.to_standard(separator='.')
            clean_text = cue.text.strip()
            # 格式：起始时间,结束时间 \n 文本 \n\n
            file_obj.write(f"{start_time},{end_time}\n{clean_text}\n\n")

    @staticmethod
    def _write_json(doc: SubtitleDocument, file_obj):
        """输出高可读性、结构化的 JSON 格式"""
        import json
        out_data = []
        for cue in doc.cues:
            out_data.append({
                # 采用数组形式存储时间戳，方便后续用索引直接调用 [0] 和 [1]
                "timestamp": [
                    cue.start.to_standard(separator='.'),
                    cue.end.to_standard(separator='.')
                ],
                "text": cue.text.strip()
            })
        # ensure_ascii=False 确保输出真实中文而非 Unicode 编码
        # indent=2 保证输出的 JSON 文件具有良好的缩进排版（漂亮！）
        json.dump(out_data, file_obj, ensure_ascii=False, indent=2)

### 🔍 为什么 JSON 里必须是 `\n`？

在 JSON 的官方规范（RFC 8259 国际标准）中，有一条铁律：**字符串（String）的内容必须写在同一行内，绝对不允许出现真实的、物理的回车换行符（Raw Newline）。**

如果你在 JSON 的两个双引号 `""` 之间直接敲一个真正的回车，整个 JSON 就会被视为**语法错误（SyntaxError）**，任何解析器（无论是浏览器的 V8 引擎，还是 Python 的 json 库）读取它时都会直接崩溃。

  "text": "这是第一行
这是第二行"
}

// ✅ 绝对标准的 JSON（你现在的代码生成的完美格式）：
{
  "timestamp": ["00:00:01.000", "00:00:03.500"],
  "text": "这是第一行\n这是第二行"
}
```

### 🛡️ 放心，这不仅不是 Bug，反而是最完美的做法

我们在代码里调用了 Python 内置的 `json.dump()`，它作为一个成熟的序列化工具，在遇到多行文本时，**会自动把内存中真实的换行符，安全地“转义”（Escape）成肉眼可见的 `\n` 字符**。

你完全不需要担心它在实际使用中会显示出 `\n` 这两个字符： 当你或者前端工程师在未来用 JavaScript、Python 或者任何语言去解析（Parse）这个 JSON 文件时，**解析引擎会在底层自动把 `\n` 变回真实的换行**。当它最终被渲染到网页（变成 `<br>`）或者视频播放器的屏幕上时，观众看到的依然是完美的两行字幕。

### 📜 LRC 规范的两大“潜规则”：

1. **绝对的单行限制（The Multiline Trap）**： LRC 是为“逐行滚动歌词”设计的。在标准 LRC 解析器眼里，**每一行都必须以 `[时间]` 开头**。 如果你把多行 SRT 字幕（比如带有 `\n` 的文本）直接塞进 LRC，第二行文本就会因为没有 `[时间]` 标签，被解析器当做无效乱码丢弃，甚至直接导致播放器报错！
2. **时间轴的“分钟进位”机制**： LRC 的标准时间格式是 `[mm:ss.xx]`（通常是两位小数的百分秒）。如果一部电影长达 1 小时 20 分钟，LRC 的时间轴**绝对不能**写成 `[01:20:00.00]`，而是必须将小时换算成分钟，写成 `[80:00.00]`。*(好消息是，咱们之前写的 `Timecode.to_lrc_format()` 方法已经完美实现了这个进位逻辑！)*

class WriterFactory:
    @staticmethod
    def write(doc: SubtitleDocument, format_type: str, out_path: str):
        with open(out_path, "w", encoding="utf-8") as f:
            if format_type == "srt":
                WriterFactory._write_srt(doc, f)
            elif format_type == "sbv":
                WriterFactory._write_sbv(doc, f)
            elif format_type == "json":
                WriterFactory._write_json(doc, f)
            elif format_type == "lrc":
                WriterFactory._write_lrc(doc, f)
            elif format_type == "txt":
                for cue in doc.cues:
                    # 纯文本模式：保留换行符，用以人类阅读
                    f.write(f"[{cue.start.to_standard()} --> {cue.end.to_standard()}] {cue.text.strip()}\n\n")

    @staticmethod
    def _write_srt(doc: SubtitleDocument, file_obj):
        """严格遵守 SubRip (.srt) 规范的输出逻辑"""
        for i, cue in enumerate(doc.cues, 1):
            start_time = cue.start.to_standard(separator=',') 
            end_time = cue.end.to_standard(separator=',')
            clean_text = cue.text.strip()
            file_obj.write(f"{i}\n{start_time} --> {end_time}\n{clean_text}\n\n")

    @staticmethod
    def _write_sbv(doc: SubtitleDocument, file_obj):
        """严格遵守 YouTube SubViewer (.sbv) 规范的输出逻辑"""
        for cue in doc.cues:
            start_time = cue.start.to_standard(separator='.')
            end_time = cue.end.to_standard(separator='.')
            clean_text = cue.text.strip()
            file_obj.write(f"{start_time},{end_time}\n{clean_text}\n\n")

    @staticmethod
    def _write_json(doc: SubtitleDocument, file_obj):
        """输出高可读性、结构化的 JSON 格式"""
        import json
        out_data = [{
            "timestamp": [cue.start.to_standard(separator='.'), cue.end.to_standard(separator='.')],
            "text": cue.text.strip()
        } for cue in doc.cues]
        json.dump(out_data, file_obj, ensure_ascii=False, indent=2)

    @staticmethod
    def _write_lrc(doc: SubtitleDocument, file_obj):
        """严格遵守 LRC 歌词规范的输出逻辑"""
        # (可选) 写入标准的 LRC ID Tags 元数据，显得更专业
        file_obj.write("[ti:Subtitle Converted by AST]\n")
        file_obj.write("[by:Super Programmer]\n")
        
        for cue in doc.cues:
            # 1. 强制将多行字幕合并为单行，用空格代替换行符，防止破坏 LRC 结构！
            # 比如 "你好\n世界" 会变成 "你好 世界"
            single_line_text = cue.text.strip().replace('\n', '  ')
            
            # 2. 标准规范：标签 [mm:ss.xx] 和文本之间通常不留多余的空格
            timestamp = cue.start.to_lrc_format()
            file_obj.write(f"[{timestamp}]{single_line_text}\n")
```

很多现代 AI 语音识别工具（比如目前最火的 **OpenAI Whisper** 模型输出的 JSON）在存储时间轴时，为了节省空间，**并不会使用带有冒号的 `HH:MM:SS` 格式，而是直接使用“纯秒数”（浮点数或整数）**。

JSON

  "timestamp": [12.5, 18.75],
  "text": "这是一句台词。"

    def from_string(cls, ts: str) -> 'Timecode':
        """将各种乱七八糟的字符串（包括纯秒数）解析为标准的时间戳对象"""
        ts_clean = (str(ts).lower()
                    .replace("h", ":").replace("m", ":")
                    .replace("时", ":").replace("分", ":")
                    .replace(",", ".")
                    .replace("s", "").replace("秒", "")
                    .strip())
        
        # 如果为空字符串，直接返回 0
        if not ts_clean or ts_clean == 'none':
            return cls(0)

        parts = ts_clean.split(":")
        
        try:
            # 🎯 修复关键点：支持没有冒号的纯秒数格式 (例如 "12.5" 或 "150")
            if len(parts) == 1:   
                h, m, s_str = 0, 0, parts[0]
            elif len(parts) == 2: # mm:ss.ms
                h, m, s_str = 0, int(parts[0]), parts[1]
            elif len(parts) == 3: # hh:mm:ss.ms
                h, m, s_str = int(parts[0]), int(parts[1]), parts[2]
            else:
                return cls(0)
            
            if '.' in s_str:
                s_val, ms_str = s_str.split('.')
                s = int(s_val)
                # 自动补齐毫秒：比如 .5 会变成 .500，.23 会变成 .230
                ms = int(ms_str.ljust(3, "0")[:3])
            else:
                s, ms = int(s_str), 0
                
            total_ms = (h * 3600 + m * 60 + s) * 1000 + ms
            return cls(total_ms)
        except ValueError:
            return cls(0)
```

            current_start = None
            current_end = None
            current_text = []

                nonlocal current_start, current_end, current_text
                if current_start and current_end and current_text:
                    doc.add_cue(current_start, current_end, "\n".join(current_text))
                current_start = current_end = None
                current_text = []

            for line in content.splitlines():
                line = line.strip()
                if not line or line.isdigit(): 
                    continue

                range_match = RANGE_PATTERN.search(line)
                if range_match:
                    flush_cue()
                    current_start = Timecode.from_string(range_match.group(1))
                    current_end = Timecode.from_string(range_match.group(2))
                    inline_text = line[range_match.end():].strip()
                    if inline_text:
                        current_text.append(inline_text)
                elif current_start:
                    current_text.append(line)
            flush_cue()

        else:
            # 匹配可能被括号包裹的单时间戳，比如 (0:00), [0:10.50] 或者直接是 0:15

### 🔍 什么时候不需要加引号？

如果你的新文件名是一连串**没有空格**的字符（比如使用了下划线、短横线或驼峰命名法），你完全可以把双引号扔掉：

### ⚠️ 什么时候必须加双引号？

双引号在命令行里的唯一作用，就是**把带有空格的多个词“打包”成一个整体参数**。

假如你想把输出的文件名命名为 `final version 2`（中间包含了空格）。如果你不加引号：

Bash

```
# ❌ 错误写法
subcomp.exe video.txt -srt -name final version 2
```

这时候命令行会犯傻，它会把 `final` 当成传递给 `-name` 的参数，然后对着后面的 `version` 和 `2` 发愣，并向你抛出一个大大的报错：“无法识别的参数”。

为了防止命令行切错词，你就必须用双引号把它们包裹起来：

Bash

```
# ✅ 正确写法（包含空格时必须加引号）
subcomp.exe video.txt -srt -name "final version 2"
```

时间轴平移（Time-shifting）绝对是做后期处理时最救命的功能！

### 🎬 什么效果？（解决什么痛点）

想象一下这个极其常见的场景：你原本已经有了一份时间轴完美对齐的字幕文件。但是，在最终定片前，你突然在片头加了 2.5 秒的 B-roll（空镜头），或者在音频后期（比如在 Studio One 里做混音时）由于采样率转换导致整体音频漂移，比原来提前了 1.2 秒。

如果没有这个功能，你要么得在非编软件里手动全选字幕往后拖（还容易拖不准），要么得重新对齐一次。 有了平移功能，你只需要在命令行输入 **`-shift 2.5`**（全体延后 2.5 秒），或者 **`-shift -1.2`**（全体提前 1.2 秒），程序就会在毫秒级别帮你把所有字幕精准平移，严丝合缝地重新贴合你的音轨和画面。

------

### 🧠 这怎么实现？（底层逻辑）

得益于咱们之前构建的 **AST（抽象语法树）架构**，实现这个功能优雅到了极致。

咱们不需要去操作那些复杂的字符串（比如去算 `00:01:59,800` 加 `1.5` 秒到底应该进位到几分几秒），因为在 AST 中，所有的起止时间都已经变成了纯粹的 `Timecode` 对象（总毫秒数）。 我们只需要：

1. 在命令行接收一个 `-shift` 的浮点数参数。
2. 遍历提取出来的所有 `SubtitleCue`（字幕节点）。
3. 调用我们早就写好的 `add_seconds()` 方法，给每一句的 `start` 和 `end` 加上这个数字。
4. 咱们的 `Timecode` 初始化里本来就有 `max(0, total_ms)` 的防弹设计，所以就算你疯狂往前移（比如 `-999` 秒），时间轴最多也就是归零，绝对不会出现导致播放器崩溃的负数时间！

class Timecode:
    """时间戳对象，内部统一使用总毫秒数存储"""
    def __init__(self, total_ms: int):
        self.total_ms = max(0, total_ms)

    @classmethod
    def from_string(cls, ts: str) -> 'Timecode':
        """将各种格式字符串（包括纯秒数）解析为时间戳对象"""
        ts_clean = (str(ts).lower()
                    .replace("h", ":").replace("m", ":")
                    .replace("时", ":").replace("分", ":")
                    .replace(",", ".")
                    .replace("s", "").replace("秒", "")
                    .strip())
        
        if not ts_clean or ts_clean == 'none':
            return cls(0)

        parts = ts_clean.split(":")
        try:
            if len(parts) == 1:   
                h, m, s_str = 0, 0, parts[0]
            elif len(parts) == 2:
                h, m, s_str = 0, int(parts[0]), parts[1]
            elif len(parts) == 3:
                h, m, s_str = int(parts[0]), int(parts[1]), parts[2]
            else:
                return cls(0)
            
            if '.' in s_str:
                s_val, ms_str = s_str.split('.')
                s = int(s_val)
                ms = int(ms_str.ljust(3, "0")[:3])
            else:
                s, ms = int(s_str), 0
                
            total_ms = (h * 3600 + m * 60 + s) * 1000 + ms
            return cls(total_ms)
        except ValueError:
            return cls(0)

    def add_seconds(self, seconds: float) -> 'Timecode':
        return Timecode(self.total_ms + int(seconds * 1000))

    def to_standard(self, separator='.') -> str:
        total_s = self.total_ms // 1000
        ms = self.total_ms % 1000
        h = total_s // 3600
        m = (total_s % 3600) // 60
        s = total_s % 60
        return f"{h:02d}:{m:02d}:{s:02d}{separator}{ms:03d}"

    def to_lrc_format(self) -> str:
        total_s = self.total_ms // 1000
        ms = self.total_ms % 1000
        m = total_s // 60
        s = total_s % 60
        xx = ms // 10
        return f"{m:02d}:{s:02d}.{xx:02d}"

TIMESTAMP_PATTERN = r'(\d{1,2}[:h时]\d{1,2}(?:[:m分]\d{1,2})?(?:[\.,]\d{1,3})?[s秒]?)'

    TIMESTAMP_PATTERN + r'\s*[\]\)\}]?'
)

class ParserFactory:
    @staticmethod
    def parse(file_path: str) -> Optional[SubtitleDocument]:
        content = ParserFactory._safe_read(file_path)
        _, ext = os.path.splitext(file_path)
        doc = SubtitleDocument()

        try:
            if ext.lower() == '.json':
                ParserFactory._parse_json(content, doc)
            elif ext.lower() == '.sbv':
                ParserFactory._parse_sbv(content, doc)
            else:
                ParserFactory._parse_text(content, doc)
            return doc if doc.cues else None
        except Exception as e:
            print(f"解析错误: {e}")
            return None

    @staticmethod
    def _safe_read(file_path: str) -> str:
        for enc in ['utf-8-sig', 'utf-8', 'gbk']:
            try:
                with open(file_path, "r", encoding=enc) as f:
                    return f.read()
            except UnicodeDecodeError: continue
        return ""

    @staticmethod
    def _parse_json(content: str, doc: SubtitleDocument):
        data = json.loads(content)
        for item in data:
            if "timestamp" in item and isinstance(item["timestamp"], list):
                start, end = str(item["timestamp"][0]), str(item["timestamp"][1])
            else:
                start = str(item.get("start_time", item.get("start", 0)))
                end = str(item.get("end_time", item.get("end", 0)))
            text = item.get("text", item.get("content", ""))
            doc.add_cue(Timecode.from_string(start), Timecode.from_string(end), text)

    @staticmethod
    def _parse_sbv(content: str, doc: SubtitleDocument):
        for block in content.split('\n\n'):
            lines = [l.strip() for l in block.split('\n') if l.strip()]
            if len(lines) >= 2:
                times = lines[0].split(',')
                if len(times) == 2:
                    doc.add_cue(Timecode.from_string(times[0]), Timecode.from_string(times[1]), "\n".join(lines[1:]))

    @staticmethod
    def _parse_text(content: str, doc: SubtitleDocument):
        if RANGE_PATTERN.search(content):
            cur_s, cur_e, cur_t = None, None, []
            for line in content.splitlines():
                line = line.strip()
                if not line or line.isdigit(): continue
                m = RANGE_PATTERN.search(line)
                if m:
                    if cur_s and cur_e and cur_t:
                        doc.add_cue(cur_s, cur_e, "\n".join(cur_t))
                    cur_s, cur_e = Timecode.from_string(m.group(1)), Timecode.from_string(m.group(2))
                    cur_t = [line[m.end():].strip()] if line[m.end():].strip() else []
                elif cur_s: cur_t.append(line)
            if cur_s and cur_e and cur_t: doc.add_cue(cur_s, cur_e, "\n".join(cur_t))

            for i, c in enumerate(temp):
                end = temp[i+1]["s"] if i+1 < len(temp) else c["s"].add_seconds(5.0)
                doc.add_cue(c["s"], end, c["t"])

                    f.write(f"{c.start.to_standard()},{c.end.to_standard()}\n{c.text}\n\n")

                    f.write(f"[{c.start.to_lrc_format()}]{c.text.replace('\\n', '  ')}\n")

                for c in doc.cues:

## Gemini 说

对于 Windows 11 的程序开发，图标（.ico）文件不仅仅是一个简单的图片，它实际上是一个**容器**，里面封装了多个不同尺寸和位深度的位图，以适应系统在各种缩放比例（DPI）和显示模式下的需求。

要制作一个标准的、专业的 Windows 11 程序图标，你需要包含以下规格：

------

### 1. 核心标准尺寸（必须包含）

为了确保图标在任务栏、桌面、资源管理器详细信息等位置都能清晰显示，你的 `.ico` 文件应至少包含以下尺寸：

| 尺寸 (像素)   | 使用场景                                         |
| ------------- | ------------------------------------------------ |
| **16 x 16**   | 资源管理器详细列表、窗口左上角小图标             |
| **32 x 32**   | 资源管理器“普通图标”视图                         |
| **48 x 48**   | 资源管理器“大图标”视图                           |
| **256 x 256** | 资源管理器“特大图标”视图、高分屏（High DPI）缩放 |

### 2. 建议增加的兼容性尺寸

为了在 Windows 11 的高分屏（200% 或更高缩放）下保持绝对锐利，建议额外加入：

- **64 x 64**
- **128 x 128**

------

### 3. 技术规范要求

- **颜色深度**：所有尺寸都应采用 **32-bit (RGBA)**。即 24 位真彩色 + 8 位 Alpha 通道（用于处理透明边框和阴影）。
- **压缩格式**：
  - **256x256** 的图标建议使用 **PNG 压缩** 存储在 `.ico` 内部（这样可以极大地减小最终生成文件的大小）。
  - 较小尺寸（16px 到 48px）通常使用非压缩的位图格式以获得最佳兼容性。

------

### 4. 推荐的图标制作流程

作为开发者，我建议你不要手动去一张张切图，效率太低且容易出错。推荐以下方案：

#### 方案 B：使用 ImageMagick (程序员最爱)

如果你电脑里装了 ImageMagick，可以用命令行一行搞定：

```

import httpx  # 推荐使用 httpx 处理异步或更复杂的请求

class SpeechProcessor:
    """语音转写处理器：对接在线 Whisper API 并生成 AST 对象"""
    
    def __init__(self, api_key: str, base_url: str = "https://api.groq.com/openai/v1"):
        self.api_key = api_key
        self.base_url = base_url

    def transcribe(self, audio_path: str, model: str = "whisper-large-v3") -> Optional[SubtitleDocument]:
        """调用 API 并将结果转换为 SubtitleDocument"""
        url = f"{self.base_url}/audio/transcriptions"
        headers = {"Authorization": f"Bearer {self.api_key}"}
        
        try:
            with open(audio_path, "rb") as f:
                files = {
                    "file": (os.path.basename(audio_path), f),
                    "model": (None, model),
                    "response_format": (None, "verbose_json"), # 获取带时间戳的详细数据
                }
                
                print(f"{TerminalColor.OKBLUE}[*] 正在上传音频并请求 Whisper 接口...{TerminalColor.ENDC}")
                response = httpx.post(url, headers=headers, files=files, timeout=600)
                response.raise_for_status()
                
                result = response.json()
                return self._to_ast(result)
                
        except Exception as e:
            print(f"{TerminalColor.FAIL}[X] API 调用失败: {e}{TerminalColor.ENDC}")
            return None

    def _to_ast(self, api_data: dict) -> SubtitleDocument:
        """核心逻辑：将 API 返回的 segments 转换为我们的 AST 节点"""
        doc = SubtitleDocument()
        
        # Whisper verbose_json 通常包含 'segments' 列表
        segments = api_data.get("segments", [])
        for seg in segments:
            # 这里的 start 和 end 通常是浮点数（秒）
            start_tc = Timecode.from_string(str(seg.get("start", 0)))
            end_tc = Timecode.from_string(str(seg.get("end", 0)))
            text = seg.get("text", "").strip()
            
            if text:
                doc.add_cue(start_tc, end_tc, text)
        
        return doc

class Timecode:
    def __init__(self, total_ms: int):
        self.total_ms = max(0, total_ms)

    @classmethod
    def from_string(cls, ts: str) -> 'Timecode':
        ts_clean = (str(ts).lower()
                    .replace("h", ":").replace("m", ":")
                    .replace("时", ":").replace("分", ":")
                    .replace(",", ".")
                    .replace("s", "").replace("秒", "")
                    .strip())
        if not ts_clean or ts_clean == 'none': return cls(0)
        
        # 兼容浮点数秒 (Whisper API 常用格式)
        try:
            return cls(int(float(ts_clean) * 1000))
        except ValueError:
            parts = ts_clean.split(":")
            try:
                if len(parts) == 1: h, m, s_str = 0, 0, parts[0]
                elif len(parts) == 2: h, m, s_str = 0, int(parts[0]), parts[1]
                elif len(parts) == 3: h, m, s_str = int(parts[0]), int(parts[1]), parts[2]
                else: return cls(0)
                
                if '.' in s_str:
                    s_val, ms_str = s_str.split('.')
                    s = int(s_val)
                    ms = int(ms_str.ljust(3, "0")[:3])
                else: s, ms = int(s_str), 0
                return cls((h * 3600 + m * 60 + s) * 1000 + ms)
            except: return cls(0)

    def add_seconds(self, seconds: float) -> 'Timecode':
        return Timecode(self.total_ms + int(seconds * 1000))

    def to_standard(self, separator='.') -> str:
        total_s = self.total_ms // 1000
        ms = self.total_ms % 1000
        h, m, s = total_s // 3600, (total_s % 3600) // 60, total_s % 60
        return f"{h:02d}:{m:02d}:{s:02d}{separator}{ms:03d}"

    def to_lrc_format(self) -> str:
        total_s, ms = self.total_ms // 1000, self.total_ms % 1000
        return f"{total_s // 60:02d}:{total_s % 60:02d}.{ms // 10:02d}"

TIMESTAMP_PATTERN = r'(\d{1,2}[:h时]\d{1,2}(?:[:m分]\d{1,2})?(?:[\.,]\d{1,3})?[s秒]?)'
RANGE_PATTERN = re.compile(r'[\[\(\{]?\s*' + TIMESTAMP_PATTERN + r'\s*[\]\)\}]?[\s\-\–\—\~>\]\[]+[\[\(\{]?\s*' + TIMESTAMP_PATTERN + r'\s*[\]\)\}]?')

class ParserFactory:
    @staticmethod
    def parse(file_path: str) -> Optional[SubtitleDocument]:
        content = ParserFactory._safe_read(file_path)
        _, ext = os.path.splitext(file_path)
        doc = SubtitleDocument()
        try:
            if ext.lower() == '.json': ParserFactory._parse_json(content, doc)
            elif ext.lower() == '.sbv': ParserFactory._parse_sbv(content, doc)
            else: ParserFactory._parse_text(content, doc)
            return doc if doc.cues else None
        except: return None

    @staticmethod
    def _safe_read(p):
        for e in ['utf-8-sig', 'utf-8', 'gbk']:
            try:
                with open(p, "r", encoding=e) as f: return f.read()
            except: continue
        return ""

    @staticmethod
    def _parse_json(c, doc):
        data = json.loads(c)
        for i in data:
            if "timestamp" in i and isinstance(i["timestamp"], list):
                s, e = str(i["timestamp"][0]), str(i["timestamp"][1])
            else:
                s, e = str(i.get("start_time", i.get("start", 0))), str(i.get("end_time", i.get("end", 0)))
            doc.add_cue(Timecode.from_string(s), Timecode.from_string(e), i.get("text", i.get("content", "")))

    @staticmethod
    def _parse_sbv(c, doc):
        for b in c.split('\n\n'):
            ls = [l.strip() for l in b.split('\n') if l.strip()]
            if len(ls) >= 2:
                ts = ls[0].split(',')
                if len(ts) == 2: doc.add_cue(Timecode.from_string(ts[0]), Timecode.from_string(ts[1]), "\n".join(ls[1:]))

    @staticmethod
    def _parse_text(content, doc):
        if RANGE_PATTERN.search(content):
            cur_s, cur_e, cur_t = None, None, []
            for line in content.splitlines():
                line = line.strip()
                if not line or line.isdigit(): continue
                m = RANGE_PATTERN.search(line)
                if m:
                    if cur_s and cur_e and cur_t: doc.add_cue(cur_s, cur_e, "\n".join(cur_t))
                    cur_s, cur_e, cur_t = Timecode.from_string(m.group(1)), Timecode.from_string(m.group(2)), [line[m.end():].strip()]
                elif cur_s: cur_t.append(line)
            if cur_s and cur_e and cur_t: doc.add_cue(cur_s, cur_e, "\n".join(cur_t))
        else:
            inline_re = re.compile(r'[\[\(\{]?\s*' + TIMESTAMP_PATTERN + r'\s*[\]\)\}]?')
            ms = list(inline_re.finditer(content))
            temp = []
            for i, m in enumerate(ms):
                t_e = ms[i+1].start() if i+1 < len(ms) else len(content)
                txt = content[m.end():t_e].strip()
                if txt: temp.append({"s": Timecode.from_string(m.group(1)), "t": txt})
            for i, c in enumerate(temp):
                end = temp[i+1]["s"] if i+1 < len(temp) else c["s"].add_seconds(5.0)
                doc.add_cue(c["s"], end, c["t"])

class WriterFactory:
    @staticmethod
    def write(doc, fmt, out_p):
        with open(out_p, "w", encoding="utf-8") as f:
            if fmt == "srt":
                for i, c in enumerate(doc.cues, 1): f.write(f"{i}\n{c.start.to_standard(',') } --> {c.end.to_standard(',')}\n{c.text}\n\n")
            elif fmt == "sbv":
                for c in doc.cues: f.write(f"{c.start.to_standard()},{c.end.to_standard()}\n{c.text}\n\n")
            elif fmt == "json":
                out = [{"timestamp": [c.start.to_standard(), c.end.to_standard()], "text": c.text} for c in doc.cues]
                json.dump(out, f, ensure_ascii=False, indent=2)
            elif fmt == "lrc":
                f.write("[ti:Converted by SubCompiler]\n")
                for c in doc.cues: f.write(f"[{c.start.to_lrc_format()}]{c.text.replace('\\n', '  ')}\n")
            elif fmt == "txt":
                for c in doc.cues: f.write(f"[{c.start.to_standard()} --> {c.end.to_standard()}] {c.text}\n\n")

class Timecode:
    def __init__(self, total_ms: int):
        self.total_ms = max(0, total_ms)

    @classmethod
    def from_string(cls, ts: str) -> 'Timecode':
        ts_clean = (str(ts).lower()
                    .replace("h", ":").replace("m", ":")
                    .replace("时", ":").replace("分", ":")
                    .replace(",", ".")
                    .replace("s", "").replace("秒", "")
                    .strip())
        if not ts_clean or ts_clean == 'none': return cls(0)
        
        # 兼容浮点数秒 (Whisper API 常用格式)
        try:
            return cls(int(float(ts_clean) * 1000))
        except ValueError:
            parts = ts_clean.split(":")
            try:
                if len(parts) == 1: h, m, s_str = 0, 0, parts[0]
                elif len(parts) == 2: h, m, s_str = 0, int(parts[0]), parts[1]
                elif len(parts) == 3: h, m, s_str = int(parts[0]), int(parts[1]), parts[2]
                else: return cls(0)
                
                if '.' in s_str:
                    s_val, ms_str = s_str.split('.')
                    s = int(s_val)
                    ms = int(ms_str.ljust(3, "0")[:3])
                else: s, ms = int(s_str), 0
                return cls((h * 3600 + m * 60 + s) * 1000 + ms)
            except: return cls(0)

    def add_seconds(self, seconds: float) -> 'Timecode':
        return Timecode(self.total_ms + int(seconds * 1000))

    def to_standard(self, separator='.') -> str:
        total_s = self.total_ms // 1000
        ms = self.total_ms % 1000
        h, m, s = total_s // 3600, (total_s % 3600) // 60, total_s % 60
        return f"{h:02d}:{m:02d}:{s:02d}{separator}{ms:03d}"

    def to_lrc_format(self) -> str:
        total_s, ms = self.total_ms // 1000, self.total_ms % 1000
        return f"{total_s // 60:02d}:{total_s % 60:02d}.{ms // 10:02d}"

TIMESTAMP_PATTERN = r'(\d{1,2}[:h时]\d{1,2}(?:[:m分]\d{1,2})?(?:[\.,]\d{1,3})?[s秒]?)'
RANGE_PATTERN = re.compile(r'[\[\(\{]?\s*' + TIMESTAMP_PATTERN + r'\s*[\]\)\}]?[\s\-\–\—\~>\]\[]+[\[\(\{]?\s*' + TIMESTAMP_PATTERN + r'\s*[\]\)\}]?')

class ParserFactory:
    @staticmethod
    def parse(file_path: str) -> Optional[SubtitleDocument]:
        content = ParserFactory._safe_read(file_path)
        _, ext = os.path.splitext(file_path)
        doc = SubtitleDocument()
        try:
            if ext.lower() == '.json': ParserFactory._parse_json(content, doc)
            elif ext.lower() == '.sbv': ParserFactory._parse_sbv(content, doc)
            else: ParserFactory._parse_text(content, doc)
            return doc if doc.cues else None
        except: return None

    @staticmethod
    def _safe_read(p):
        for e in ['utf-8-sig', 'utf-8', 'gbk']:
            try:
                with open(p, "r", encoding=e) as f: return f.read()
            except: continue
        return ""

    @staticmethod
    def _parse_json(c, doc):
        data = json.loads(c)
        for i in data:
            if "timestamp" in i and isinstance(i["timestamp"], list):
                s, e = str(i["timestamp"][0]), str(i["timestamp"][1])
            else:
                s, e = str(i.get("start_time", i.get("start", 0))), str(i.get("end_time", i.get("end", 0)))
            doc.add_cue(Timecode.from_string(s), Timecode.from_string(e), i.get("text", i.get("content", "")))

    @staticmethod
    def _parse_sbv(c, doc):
        for b in c.split('\n\n'):
            ls = [l.strip() for l in b.split('\n') if l.strip()]
            if len(ls) >= 2:
                ts = ls[0].split(',')
                if len(ts) == 2: doc.add_cue(Timecode.from_string(ts[0]), Timecode.from_string(ts[1]), "\n".join(ls[1:]))

    @staticmethod
    def _parse_text(content, doc):
        if RANGE_PATTERN.search(content):
            cur_s, cur_e, cur_t = None, None, []
            for line in content.splitlines():
                line = line.strip()
                if not line or line.isdigit(): continue
                m = RANGE_PATTERN.search(line)
                if m:
                    if cur_s and cur_e and cur_t: doc.add_cue(cur_s, cur_e, "\n".join(cur_t))
                    cur_s, cur_e, cur_t = Timecode.from_string(m.group(1)), Timecode.from_string(m.group(2)), [line[m.end():].strip()]
                elif cur_s: cur_t.append(line)
            if cur_s and cur_e and cur_t: doc.add_cue(cur_s, cur_e, "\n".join(cur_t))
        else:
            inline_re = re.compile(r'[\[\(\{]?\s*' + TIMESTAMP_PATTERN + r'\s*[\]\)\}]?')
            ms = list(inline_re.finditer(content))
            temp = []
            for i, m in enumerate(ms):
                t_e = ms[i+1].start() if i+1 < len(ms) else len(content)
                txt = content[m.end():t_e].strip()
                if txt: temp.append({"s": Timecode.from_string(m.group(1)), "t": txt})
            for i, c in enumerate(temp):
                end = temp[i+1]["s"] if i+1 < len(temp) else c["s"].add_seconds(5.0)
                doc.add_cue(c["s"], end, c["t"])

class WriterFactory:
    @staticmethod
    def write(doc, fmt, out_p):
        with open(out_p, "w", encoding="utf-8") as f:

                for i, c in enumerate(doc.cues, 1): f.write(f"{i}\n{c.start.to_standard(',') } --> {c.end.to_standard(',')}\n{c.text}\n\n")

                for c in doc.cues: f.write(f"[{c.start.to_lrc_format()}]{c.text.replace('\\n', '  ')}\n")
            elif fmt == "txt":
                for c in doc.cues: f.write(f"[{c.start.to_standard()} --> {c.end.to_standard()}] {c.text}\n\n")

## Gemini 说

这是一个非常深刻的工程问题。Whisper 这种基于 Transformer 架构的模型，其本质是一个“概率预测机器”。当音频进入结尾、静音或背景噪音较大时，模型的信噪比下降，它就会开始根据训练数据（大量的 YouTube 视频、电视剧字幕）进行“脑补”，从而产生你看到的“优优独播剧场”这类幻觉。

要通过 **Prompt（提示词）** 来约束 AI，使其“只听不编”，核心策略是**通过 Prompt 锚定语境，降低模型发散的概率**。

------

### 🛠️ 如何调整 Prompt 策略

在 Whisper API 的请求中，`prompt` 参数非常特殊：它不像是 ChatGPT 那样的指令，而更像是一段**“前情提要”**或**“风格示例”**。

#### 1. 策略一：使用“中立语境”锚点

给模型一个极其枯燥、学术或正式的开头，让模型认为接下来的内容是严谨的录音，而不是娱乐视频。

- **推荐 Prompt**: `“以下是一段正式的会议记录或学术讲座录音，内容严谨，不包含任何背景音乐、广告或频道推广语。”`
- **原理**：这会提高模型输出“正式词汇”的权重，降低其输出“剧场”、“订阅”等娱乐词汇的概率。

#### 2. 策略二：使用“标点符号”暗示

Whisper 的 Prompt 如果包含正确的标点符号，会引导模型也产出结构整齐的文本，减少胡言乱语。

- **推荐 Prompt**: `“你好。这是一段清晰的对话。请准确记录每一个字，不要添加无关信息。”`

#### 3. 策略三：利用“关键词避坑”

如果你发现它总是输出特定的幻觉，可以在 Prompt 里反向操作，虽然 Whisper 对“不要做什么”的指令理解力不如 GPT，但提供正确的术语会有帮助。

- **推荐 Prompt**: `“这段音频关于阿德勒心理学、哲学讨论。请仅转录人声对话。”`（针对你之前处理的《被讨厌的勇气》内容）。

### 🧠 深度优化建议（双管齐下）

单靠 Prompt 有时可能无法 100% 消除顽固的幻觉（尤其是音频末尾长达数秒的静音时）。建议你配合以下**工程手段**：

1. **静音切除 (VAD)**： 在上传音频前，使用 Python 的 `pydub` 或 `webrtcvad` 库切掉音频开头和结尾的静音部分。Whisper 只要听不到“虚无”，就不会在虚无中“创造”优优剧场。
2. **温度系数 (Temperature)**： 如果 API 支持，将 `temperature` 设置为 `0`。
   - `Temperature = 0`：AI 变得死板、保守，只选概率最高的词（也就是它听到的声音）。
   - `Temperature = 0.8`：AI 变得有创意（这就是产生幻觉的温床）。
3. **置信度过滤**： 在 `verbose_json` 返回的数据中，每个 segment 其实都有一个 `avg_logprob`（平均对数概率）。幻觉内容的这个分值通常很低。你可以写一行代码： `if seg.get("avg_logprob", 0) < -1.0: continue` 直接丢弃那些 AI 自己都觉得是“瞎编”的句子。
